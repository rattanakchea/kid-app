# Resume Framing for Emoji Learning Flashcard Game for Kids

## Summary

Position this project as an end-to-end product build where you defined scope, shipped a browser-first educational app, extended it into an iOS-ready product with Capacitor, prepared App Store assets/compliance work, and used agentic AI to accelerate product, engineering, and release execution.

## Resume Copy

Use this as the primary resume entry:

**Full-Stack / Product Engineer, Emoji Learning Flashcard Game for Kids**

- Built and shipped a browser-first educational game for young children using React, TypeScript, and Vite, delivering flashcards, audio-assisted vocabulary learning, and pair-matching gameplay across themed content packs such as First Words, Animals, Fruits, and Colors.
- Extended the product to iOS with Capacitor and native Swift plugin work, including offline-friendly asset packaging and one-time premium unlock plumbing via StoreKit for future in-app purchases.
- Designed and iterated on a production-minded solo-founder workflow: defined MVP scope, broke work into safe increments, maintained documentation and release checklists, and connected GitHub to Vercel for automated preview/production deployments.
- Improved product polish for mobile and App Store readiness by refining responsive card layouts, creating iPhone/iPad screenshot assets, fixing icon transparency issues, and finalizing submission-facing branding and metadata.
- Used agentic AI as a development copilot for planning, implementation, debugging, documentation, and release preparation while keeping architecture simple, maintainable, and reviewable.

## Tech Stack

Use this stack line:

- React, TypeScript, Vite, Capacitor iOS, Swift, StoreKit, Vitest, Testing Library, Vercel, GitHub, Xcode, agentic AI-assisted development

## Key Challenges

Use these challenge bullets in interviews, project notes, or a portfolio:

- Balancing fast MVP delivery with App Store expectations for polish, assets, metadata, and native packaging
- Adapting a web-first React app into an iOS-ready product through Capacitor without overbuilding backend infrastructure
- Designing a simple architecture that supported multiple learning modes, content packs, offline-friendly assets, and future premium unlocks
- Resolving native release issues such as non-transparent iOS app icons, screenshot formatting, and submission-readiness details
- Using agentic AI effectively as a force multiplier while still enforcing small safe increments, documentation discipline, and human review

## Assumptions

- Optimize for a full-stack/product-engineer role rather than a pure iOS-native role
- Emphasize shipped outcomes, release engineering, and product ownership more than low-level algorithmic complexity
- Keep “agentic AI” as an accelerator in the story, not the main accomplishment
