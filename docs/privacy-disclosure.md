# App Privacy Draft

## Intended v1 posture

- No third-party advertising SDKs
- No third-party analytics SDKs
- No account creation required
- No user profile collection for the core game loop
- Web preview may store premium unlock state in local browser storage
- iOS premium entitlement state is determined by Apple StoreKit transactions

## Before submission

- Reconcile App Store Connect privacy answers against the final shipped SDK list
- Do not claim "no data collected" unless all implementation paths support that claim
- Replace placeholder support and privacy URLs with production HTTPS pages
